Agent Harness Control Taxonomy

エージェントを「なんとなく安全に動かす」のではなく、どの段階で、どんな種類の制御を置くのかを分けて考えるための整理。レビュー、実行、文脈、権限の境界を混ぜないための読み物です。

1. 制御は一枚岩ではない実行前、実行中、実行後で効く仕組みが違う。全部を「ガードレール」と呼ぶと設計がぼやける。
2. 文脈と安全を分けるrules、memory、skillsは文脈ハーネス。sandbox、approval、auditは安全ハーネス。混ぜると責任範囲が見えなくなる。

今日のテーマは「制御の名前を分ける」こと

AIエージェントの運用では、いろいろな仕組みがまとめて「ハーネス」や「ガードレール」と呼ばれがちです。ルールファイル、承認フロー、テスト、メモリ、サンドボックス、レビュー、監査ログ。どれも重要ですが、同じ種類の制御ではありません。

Agent Harness Control Taxonomy は、この混ざりやすい領域に名前を与えるためのノートです。制御を「いつ効くのか」と「どう効くのか」に分けると、設計の会話がかなり楽になります。

実行前、実行中、実行後で役割が違う

まず時間軸があります。実行前の制御は、エージェントが何を見るか、何を目標にするか、どの手順を踏むかを決めます。AGENTS.md、skills、テンプレート、タスク定義、ルールファイルはここに入ります。

実行中の制御は、ツール使用、状態管理、リトライ、承認、サンドボックスのように、エージェントが動いている最中の振る舞いを縛ります。危ない操作を止める、途中で人間確認を挟む、権限を限定する、といった役目です。

実行後の制御は、テスト、lint、レビュー、差分確認、公開URLのHTTP確認などです。成果物が本当に使えるか、外に出してよいか、次回に残すべき学びがあるかを見る層です。

決定的な制御と意味的な制御

もうひとつの軸は、制御が機械的に決まるか、意味判断を含むかです。型チェックやJSON構文確認は決定的です。通るか落ちるかが明確で、安く何度でも回せます。

一方で、「読み物として自然か」「ユーザーの意図に合っているか」「公開して問題ない粒度か」は意味的な判断です。これはrubric、レビュー、別エージェント、人間確認の出番になります。

大事なのは、どちらか一方に寄せないことです。安い機械チェックで落とせる問題は先に落とし、意味判断が必要なところに人間や評価エージェントの注意を残す。これがレビューを詰まらせない基本形です。

評価、実行、文脈、安全を混ぜない

同じ「ハーネス」でも、評価ハーネス、実行ハーネス、文脈ハーネス、安全ハーネスは別物です。評価ハーネスは成果物やモデル出力を測る。実行ハーネスはツール利用やループを回す。文脈ハーネスはrules、memory、skills、MCPなどでモデルに何を見せるかを決める。安全ハーネスは権限、承認、監査、サンドボックスを担当します。

ここを混ぜると、対策がずれます。たとえば、メモリに「危ない操作をしない」と書くだけでは安全境界にはなりません。逆に、サンドボックスだけ強くしても、良い読み物を作る文脈設計にはなりません。

今回のPodcast運用にもそのまま効く

今回の変更は小さく見えて、実はこの分類がかなり効いています。Masakiが求めていたのはNotebookLMの認証や音声生成ではなく、毎回読める reading.html を出すことでした。

ここで必要だった制御は、NotebookLM authの復旧ではありません。出力仕様の修正です。つまり文脈ハーネスと実行後レビューの問題でした。「読み物」と言ったときにsource.mdのHTML化ではなく、専用の読み物版を生成する。これをCron手順と投稿スクリプトに入れるのが正しい場所です。

失敗は、次の手順に昇格させる

エージェント運用で一番もったいないのは、人間が一度指摘したことを次回また忘れることです。今回なら「読み物とはreading.htmlのこと」という指摘は、会話の中だけに置くと消えます。

だから、定期ジョブの手順、投稿URL、スキル提案、メモリに落とします。これで次回から、同じ言葉を聞いたときの解釈が変わります。ハーネスは大げさな基盤だけではなく、こういう小さな補正の積み重ねでもあります。

「ガードレールを置く」では足りない。どの段階に、どの種類の制御を置くのかを分けると、エージェント運用の改善点が見えやすくなる。

次に考えるとよさそうな問い

  • いまの自動化で、実行前・実行中・実行後の制御はそれぞれどこにあるか？
  • 文脈で縛るべきことと、権限や承認で縛るべきことを混ぜていないか？
  • レビューで出た指摘を、次回のskillやCron手順に昇格できているか？
  • 「読み物」「公開」「完了」などの言葉は、成果物レベルで明確になっているか？
